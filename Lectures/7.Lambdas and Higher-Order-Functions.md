# Lambdas and Higher-Order-Functions
## Thu Nov 27 

- Higher-order functions in java : 
    - Java doesn't have first-class or higher-order dfunctions 

- Functional interfaces: 
    - consumer is an interface from the java.util.fucntion package
    - consumer is a special kind of interface called functional interface
    - Functional interfaces is any interface with only one abstract method 
        - Ex: public interface MyInterface {
            void myMethod();
            default void myDefaultMethod() {
                System.out.println("My default method");
            }
        }

    - @FunctionalInterface annotation can be used to enable compile-time warnings if an interface isn't functional

- Implementing functional interfaces
    - can be implemented without writing a full class
    - Lambda expressions can be used
        - ex: @FunctionalInterface
               public interface UnaryMathOp {
                double calc(double x);
               }
               MathOp double = x -> x*2

- Using lambda expressions: 
    - a lambda expression can be used instead of a functional interface
        - Ex: public void doMath(double x, double y, MathOp op) {
            System.out.println(op.apply(x,y));
        }
        doMath(5, 10, (x,y)-> x * y);

- GUI event handling: 
    - ex: button.addActionListener(e -> {
        // Do something
    });
    - addActionListener accepts an ActionListener which is a function interface

- Higher-order functions in Java
    - public void forEach(Consumer<T> action)
    - list.forEach(item -> System.out.println(item));

- Function references: 
    - referenced fucntion must match the fuctional interface abstract method signature
    - list.forEach(item -> System.out.println(item));
    - list.forEach(System.out::println);

- Higher-order fuctions in java:
    - Lambda expressions and fucntion reference are java way of allowing for higher-order fuction-like syntax
    - Both are implicity converted to Anonymous Inner Classes

- Anonymous Inner Classes: 
    - is a one-time, unnamed implementation of an interface that is usually immediately instantiated
